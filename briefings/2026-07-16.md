## 摘要

今日开源社区围绕 AI 工作流自动化和 AI 内容创作两个方向集中发力，涌现出一批具备生产级能力的工具与框架。工作流侧强调多智能体编排、MCP 协议集成和浏览器自动化；内容创作侧则聚焦在从文本到动画短剧的全流程生成，以及融合多个大模型以提升创作质量。

## AI 工作流自动化

该方向以公平代码或开源工具为核心，将 AI 能力融入可视化工作流与多代理协作中。项目普遍强调自托管、私有化部署以及对 LLM API 的直连，降低对外部平台的依赖，使自动化脚本能够安全、灵活地扩展到数百种集成。

● **[n8n-io/n8n](https://github.com/n8n-io/n8n)**｜作者：n8n-io
  该项目主要实现了：公平代码的 AI 工作流自动化平台，支持可视化构建与自定义代码，原生集成 AI 能力，提供 400 多个连接器，可自托管或云部署。
  亮点：将低代码工作流与 AI 原生融合，极大降低了复杂自动化场景的搭建门槛；庞大的集成生态使其几乎能串接所有主流工具。

● **[activepieces/activepieces](https://github.com/activepieces/activepieces)**｜作者：activepieces
  该项目主要实现了：面向 AI 代理与 MCP 协议的工作流自动化平台，内置约 400 个 MCP 服务器，让 AI 代理直接调用各种服务，实现智能决策和多步自动化。
  亮点：深度集成 MCP 协议，让工作流中的 AI 代理具备即插即用的工具调用能力，大幅拓展了代理可操作的范围。

● **[nanobrowser/nanobrowser](https://github.com/nanobrowser/nanobrowser)**｜作者：nanobrowser
  该项目主要实现了：开源 Chrome 扩展，利用用户自己的 LLM API 密钥驱动多代理网页自动化，可替代 OpenAI Operator 等商业产品。
  亮点：完全开源且通过本地 API 密钥运行，避免了厂商锁定和隐私泄露风险；多代理协作架构能完成复杂的跨页面自动化任务。

## AI 内容创作

从长篇叙事到短剧视频，AI 正以“作者代理”或“混合模型”的方式介入创作的各个环节。这类工具不仅降低动画制作的门槛，还通过融合多个大模型来平衡创意与执行力，服务于 IP 开发、脚本生成和交互式内容。

● **[HBAI-Ltd/Toonflow-app](https://github.com/HBAI-Ltd/Toonflow-app)**｜作者：HBAI-Ltd
  该项目主要实现了：一站式 AI 短剧创作桌面应用，能将小说或剧本快速转化为动画短剧，集成 AI 编剧、智能分镜、角色与视频生成模块。
  亮点：以端到端的自动化消除了短剧制作中繁琐的手工环节，让创作者可以批量、低成本地产出视觉内容，极大缩短从文本到成片的时间。

● **[Narcooo/inkos](https://github.com/Narcooo/inkos)**｜作者：Narcooo
  该项目主要实现了：故事创作 AI 代理，覆盖小说、剧本、翻译、互动游戏和 IP 开发等多种叙事形式，以单一代理引擎支撑跨体裁的创意工作。
  亮点：将多样化叙事需求统一在一个代理框架内，尤其突出互动游戏和 IP 内容生成，为交互式叙事提供了原生的 AI 创作支撑。

● **[ErlichLiu/DeepClaude](https://github.com/ErlichLiu/DeepClaude)**｜作者：ErlichLiu
  该项目主要实现了：通过组合 DeepSeek r1 与 Claude 3.7 Sonnet 或 Gemini 2.5 Pro 等模型，提供针对代码生成和内容创作的超高性价比推理方案，并兼容 OpenAI 接口。
  亮点：用“取长补短”的多模型组合策略，在代码和内容两个场景中都实现了比单一模型更优的质量，且完全兼容现有生态的开源实现。
